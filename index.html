RR-TM: Improving Concurrency for
Eager Software Transactional Memory

Gaurav Jain and Patrick Lam

University of Waterloo

December 2, 2013

Situation


  atomic 
  {
    process(n4);
  }

implies the following operations:


  atomic {
    READ head
    READ n1.next
    READ n2.next
    READ n3.next
    process(n4)
  }

Contention

Let's throw in a WRITE.


  atomic {
    WRITE n1.next
  }
	      

  atomic {
    READ head
    READ n1.next
    READ n2.next
    READ n3.next
    process(n4)
  }

Eager STM = unmediated WRITEs; go directly to memory.

READer can stall or abort.
